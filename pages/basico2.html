Arrays

Los arrays son objetos similares a una lista cuyo prototipo proporciona métodos para efectuar operaciones de recorrido y de mutación. Tanto la longitud como el tipo de los elementos de un array son variables. Dado que la longitud de un array puede cambiar en cualquier momento, y los datos se pueden almacenar en ubicaciones no contiguas, no hay garantía de que los arrays de JavaScript sean densos; esto depende de cómo el programador elija usarlos. En general estas características son cómodas, pero si, en su caso particular, no resultan deseables, puede considerar el uso de arrays con tipo.

Estructura:
let arraysEjemplo = [true, 'Buenos Aires', 13];
console.log(arraysEjemplo)
OUTPUT:
true,'BuenosAires',13

En el lugar del valor hay unos corchetes dentro de los cuales los valores irán separados por coma. Los valores aceptados son todos. Los valores reciben un valorr numerico dependiendo del orden en que establecimiento de los mismos, siendo el 0, el valor correspondiente al primer lugar. En el ejemplo anterior true es el primer valor correspondiendo con 0.

Acceder a valores

Para solicitar los valores que hay dentro de un array se usa la siguiente estructura:
console.log(arraysEjemplo[0]);
OUTPUT:
true
Nota: Si ingreso un número superior a la lóngitud del array tendre por resultado el valor "undefined". Todas las posiciones posibles que no reciban un valor serán "undefined". Arrays maneja posicion que empieza desde 0 que es diferente al número de elemento. Se dice que los datos estan en un indice = posicion.

Arrays convertidos a objetos

Un array se convertira en objeto si una de sus propiedades recibe como indice una cadena de texto y por lo tanto los metodos que antes funcionaban con los arrays tenderán a no funcionar correctamente como en el caso de .length

Arrays asociativos

Un array asociativo es aquel en el que cada elemento no está asociado a su posición numérica dentro del array, sino que está asociado a otro valor específico. Es decir, una posición podría estar asociada a otra y en consecuencia cuando se llame esa primera posición se llamaría la posición que esta asociada, por ejemplo: un array[0] que tiene asociado el indice 1 siempre dara como resultado array[1]

Ej:
let asoc = {
    nombre: 'Jhon',
    lastName: 'Cena'
};
console.log(asoc);
console.log(asoc[0]);
console.log(asoc['nombre']);
OUTPUT: [object Object]
undefined
'Jhon'

Bucles e iteración

Los bucles ofrecen una forma rápida y sencilla de hacer algo repetidamente.

while

Una declaración while ejecuta sus instrucciones siempre que una condición especificada se evalúe como true.
Si la condición se vuelve false, la instrucción dentro del bucle se deja de ejecutar y el control pasa a la instrucción que sigue al bucle.

Ej:
let n = 6;
while(n > 3) {
    console.log('Repetición');
    n--;
}

do...while

La instrucción do...while se repite hasta que una condición especificada se evalúe como falsa.
Si condición es true, la declaración se ejecuta de nuevo. Al final de cada ejecución, se comprueba la condición. Cuando la condición es false, la ejecución se detiene y el control pasa a la declaración que sigue a do...while.

Ej:
let n = 7
do {
n++;
console.log('Repetición');
} while(n < 10)

for

Crea un bucle que consiste en tres expresiones opcionales, encerradas en paréntesis y separadas por puntos y comas, seguidas de una sentencia ejecutada en un bucle.
Exprecion inicial(declaración):
Una expresión (incluyendo las expresiones de asignación) o la declaración de variable. Típicamente se utiliza para usarse como variable contador. Esta expresión puede opcionalmente declarar nuevas variables con la palabra clave var. Estas variables no son locales del bucle, es decir, están en el mismo alcance en el que está el bucle for. El resultado de esta expresión es descartado.
Condición:
Una expresión para ser evaluada antes de cada iteración del bucle. Si esta expresión se evalúa como verdadera, se ejecuta sentencia. Esta comprobación condicional es opcional. Si se omite, la condición siempre se evalúa como verdadera. Si la expresión se evalúa como falsa, la ejecución salta a la primera expresión que sigue al constructor de for.
Sentencia o iteración:
Una sentencia que se ejecuta mientras la condición se evalúa como verdadera. Para ejecutar múltiples sentencias dentro del bucle, utilice una sentencia block ({ ... }) para agrupar aquellas sentecias.

for(let i = 0; i < 6 ; i++) {
    console.log(i);
}

Sentencia break

Termina el bucle actual, sentecia switch o label y transfiere el control del programa a la siguiente sentencia a la sentecia de terminación de éstos elementos.

Ej:
let num = 10;
while(num > 1) {
    num--;
    console.log(num)
    if(num == 5) {
        break;
    }
}

Nota: Adentro de la condición puede ir cualquier variable, no hace falta declararla, solo mencionarla.

Sentencia continue

for(let i = 10; i > 0; i--) {
    if( i == 5 ) {         continue;
    }
    console.log(i);
}

for...in

La instrucción for-in itera sobre todas las propiedades enumerables de un objeto que está codificado por cadenas (ignorando los codificados por Símbolos, incluidas las propiedades enumerables heredadas. Luego del flor irá una palabra clave que representara un valor individual por cada iteración. El primer valor de for devuelve el indice de los elementos.

Ej1:
let lengs = ['js', 'python', 'php', 'html', 'css'];
for(leng in lengs) {
    console.log(lengs[leng]);
}
OUTPUT:
'js'
'python'
'html'
'css'

Ej2:
let lengs = ['js', 'python', 'php', 'html', 'css'];
lengs.otro = 'C'; for(leng in lengs) {
    console.log(leng);
}
OUTPUT:
0
1
2
3
4
otro

for...of

La sentencia sentencia for...of ejecuta un bloque de código para cada elemento de un objeto iterable, como lo son: String, Array, objetos similares a array (por ejemplo, arguments or NodeList), TypedArray, Map, Set e iterables definidos por el usuario. El primer valor de for devuelve el valor de los elementos.

Ej:
let lengs = ['js', 'python', 'php', 'html', 'css'];
lengs.otro = 'C';
for(leng in lengs) {
    console.log(leng);
}
OUTPUT:
'js'
'python'
'html'
'css'
'C'

Label

Proporciona una sentencia con un identificador al que se puede referir al usar las sentencias break o continue.
Por ejemplo, puede usar una etiqueta para identificar un bucle, y entonces usar las sentencias break o continue para indicar si un programa debería interrumpir el bucle o continuar su ejecución.
Sintaxis:
etiqueta :sentencia

Ej:
let nombresUno = [ 'exilon', 'zetha', 'sigma' ];
let nombresCero =[ 'alpha', 'beta', 'gamma', nombresUno ];
forNombres:
for( let nombre in nombresCero ) {
    if( nombre == 3) {
        for(let nombre of nombres) {
           console.log(nombre);
           break forNombres;         }
    } else {
        console.log(nombresCero[nombre]);
    }
}
 Nota: Una vez alcanzado el label forNombres dentro del algoritmo terminara el ciclo completo ya que el nombre del label representa el ciclo entero. En caso de que en vez de que hubiese un break hubiese un continue, se saltaría el bucle completo.

Funciones

Las funciones son uno de los bloques de construcción fundamentales en JavaScript. Una función en JavaScript es similar a un procedimiento — un conjunto de instrucciones que realiza una tarea o calcula un valor, pero para que un procedimiento califique como función, debe tomar alguna entrada y devolver una salida donde hay alguna relación obvia entre la entrada y la salida. Para usar una función, debes definirla en algún lugar del ámbito desde el que deseas llamarla.

Sintaxis & Declaración de funcion:
Una definición de función (también denominada declaración de función o expresión de función) consta de la palabra clave function, seguida de:
El nombre de la función.
Una lista de parámetros de la función, entre paréntesis y separados por comas.
Las declaraciones de JavaScript que definen la función, encerradas entre llaves, { ... }.

function square(number) {
     return number * number;
}

 La función square toma un parámetro, llamado number. La función consta de una declaración que dice devuelva el parámetro de la función (es decir, number) multiplicado por sí mismo. La instrucción return especifica el valor devuelto por la función:

Llamar funciones

Definir una función no la ejecuta. Definirla simplemente nombra la función y especifica qué hacer cuando se llama a la función.
Llamar a la función en realidad lleva a cabo las acciones especificadas con los parámetros indicados. Por ejemplo, si defines la función square, podrías llamarla de la siguiente manera:

square(5);



La declaración anterior llama a la función con un argumento de 5. La función ejecuta sus declaraciones y devuelve el valor 25.
Las funciones deben estar dentro del ámbito(scope) cuando se llaman, pero la declaración de la función se puede elevar (cuando aparece debajo de la llamada en el código), como en este ejemplo:

console.log(square(5));
/* ... */
function square(n) { return n * n }

return

La sentencia return finaliza la ejecución de la función y especifica un valor para ser devuelto a quien llama a la función. "return" entonces sirve para devolver un tipo de dato que puede ser almacenado con posterioridad en otra variable por ejemplo. En caso de no usar return dicho proceso no sucedera. Se puede diferenciar entonces entre lo que "hace" la función y lo que retorna.

Parámetros & scope

El parámetro de una función son aquellos valores que serán utilizados dentro de la funcionalidad de la función o del return y que pueden ser cualquier valor, van entre parentesis antes del bloque funcional. Utilizar un parametro en una funcion es igual a declarar las variables a utilizar en las mismas, y usar la funcion es igual a inicializar las variables dentro de la función. Estas variables solo podrán ser usadas dentro del SCOPE de la función.

Lás variables creadas dentro del bloque de la funcion solo tendran alcanze local y por lo tanto no podrán ser usadas desde afuera de la misma a menos de que se empleen "var" o ninguna palabra en absoluto para declarar la variable(equivalente a var), de esa manera, la variable será global.

Los parámetros valores que recibiran los parámetros se llaman argumentos. Los argumentos dentro de la función(siempre que sean más de uno) pueden ser llamados como si fueran arrays. Ej: arguments[0] arguments[1].

Si no se pasaran argumentos suficientes para una funcion y esta es llamada, los valores por defecto serán undefined

Funcion anónima

Si bien la declaración de función anterior sintácticamente es una declaración, las funciones también se pueden crear mediante una expresión function.
Esta función puede ser anónima; no tiene por qué tener un nombre. Por ejemplo, la función square se podría haber definido como:

const square = function(number) { return number * number }
var x = square(4);

Sin embargo, puedes proporcionar un nombre con una expresión function. Proporcionar un nombre permite que la función se refiera a sí misma y también facilita la identificación de la función en el seguimiento de la pila de un depurador:

const factorial = function fac(n) { return n < 2 ? 1 : n * fac(n - 1) }
console.log(factorial(3))

Función flecha

Una expresión de función flecha es una alternativa compacta a una expresión de función tradicional, pero es limitada y no se puede utilizar en todas las situaciones.

Diferencias y limitaciones

  • No tiene sus propios enlaces a this o super y no se debe usar como métodos.

  • No tiene argumentos o palabras clave new.target.

  • No apta para los métodos call, apply y bind, que generalmente se basan en establecer un ámbito o alcance

  • No se puede utilizar como constructor.

  • No se puede utilizar yield dentro de su cuerpo.

Sintaxis:

Ej1:
const saludar = (nombre) => {
    console.log(nombre);
}

Ej2(solo si hay un parámetro):
const saludar = nombre => {
    console.log(nombre);
}

Ej3(solo si hay un parámetro y con un parentesis):
const saludar = (nombre => {
    console.log(nombre);
}

Ej4(Cuando en el bloque solo hay una línea de código pueden obviarse las llaves):
const saludar = (nombre) => console.log(nombre);

Ej5(full resumida):
const saludar = nombre => console.log(nombre)

Nota: No hace falta usar return ya que es como si estubiera implicito.
