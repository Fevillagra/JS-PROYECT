Programación orientada a objetos

OOP (Programación orientada a objetos) es un paradigma de programación en el que los datos son encapsulados en objetos, los cuales tienen su propio comportamiento.
JavaScript esta altamente orientado a objetos. Sigue el modelo basado en prototipado (en oposición al modelo basado en clases).

Objeto

Un objeto es una colección de datos relacionados y/o funcionalidad (que generalmente consta de algunas variables y funciones, que se denominan propiedades y métodos cuando están dentro de objetos).

Conceptos básicos

Clases

Definiendo clases Las clases son "funciones especiales", como las expresiones de funciones y declaraciones de funciones, la sintaxis de una clase tiene dos componentes: expresiones de clases y declaraciones de clases. Las clases de JavaScript proveen una sintaxis mucho más clara y simple para crear objetos y lidiar con la herencia.

Metodos

El valor de una propiedad puede ser una función, en cuyo caso la propiedad es conocida como un método. Nota:

Constructor

Al crearse la clase es necesario crear también un constructor que creara las propiedades del objeto.

instanciación

Cuando la clase es creada se dice que está "instanciada"

Ej:

Creado clases:

class animal {
    constructor(especie,edad,color) {
        this.especie = especie;
        this.edad = edad;
        this.color = color;
        this.info = `Esta animal es de la especie ${this.especie}, de color ${color}, y ${edad} año/s.`     }
    verInfo() {
        console.log(this.info);
    }
}
 Nota: Cuando se establece el this.algo estoy referenciando al objeto que voy a crear, ,es decir, estoy creando sus propiedadesmientras que especie, por ejemplo son los parámetros de los valores que alojaremos dentro de las propiedades del objeto.

Instanciando Objeto

let gato = new animal("perro", 5, "rojo");
 Nota: Si el valor de las propiedades no es especificado el valor por defecto será "undefined", aunque las propiedades seguiran existiendo.

Accediendo a propiedades & metodos

INPUT:
console.log(perro.color);
OUTPUT:
"rojo"
 Nota: Para acceder a cualquier propiedad será necesario usar un punto seguido del nombre de la propiedad.
INPUT:
perro.verInfo();
OUTPUT:
"Esta animal es de la especie perro, de color rojo, y 5 año/s."

El metodo puede ser creado fuera de la propiedad, pero nombrado como valor dentro de una propiedad.
EJ:
INPUT:
function animal(tipo){
    this.tipo = tipo;
    this.sonido = hablar;
}
function hablar() {
    console.log('Waw!');
}
let perro = new animal('perro');
perro.sonido();
OUTPUT:
'Waw!'

Abstracción

En programación, una abstracción es una manera de reducir la complejidad y permitir un diseño e implementación más eficientes en sistemas de software complejos. En programación orientada a objetos es la simplificación de un objeto.

Modularidad

Es conveniente simplificar la escritura de codigo en múltiples soluciones en vez de crear una solucion muy grande que sea demasiado abarcativa en terminos del problema que estoy solucionando debido a que un solo error en el codigo podría costarnos todo el trabajo conseguido, en cambio si simplifico la solucion en multiples "partes" si hay un solo error, solo tendré que correjir la "parte" en la que está el error.Un módulo es una porción de un programa de ordenador. De las varias tareas que debe realizar un programa para cumplir con su función u objetivos, un módulo realizará, comúnmente, una de dichas tareas (o varias, en algún caso)

Encapsulado

El encapsulado es el ocultamiento de información sensible a la que se le priva al usuario el acceso.

Polimorfismo

Es la cualidad de un objeto de reaccionar con diferencia a otro objeto ante la ejecución de un metodo.

Herencia

La herencia es la creación de una clase que funcione igual que una clase diferente la establecida con la diferencia de que se le puede agregar propiedades y funcionalidades extra.

Ej:
class perro extends animal {
    constructor(especie,edad,color,raza) {
        super(especie,edad,color);
        this.raza = raza;
    }
    ladrar() {
        alert('Waw!');
}
}

let dogo = new perro('perro', 5, negro, 'sharpey');
 Nota: No puedo tener una clase con el mismo nombre que el objeto.

El perro heredara todas las propiedades de animal y adicionalmente tendrá la propiedad raza y el metodo ladrar.

Metodos estáticos

Un metodo estático es un metodo que no necesita que la clase se defina para poder ser creado. Para ello se usa la palabra clave "static"

Ej:
class perro extends animal {
    constructor(especie,edad,color,raza) {
        super(especie,edad,color);
        this.raza = raza;
    }
    static ladrar() {
        alert('Waw!')
}
}
perro.ladrar();
OUTPUT:
'Waw!'
Nota: Puede llamarse directamente el metodo aunque el objeto no exista ya que el metodo es estático. si el metodo referenciara alguna propiedad dicha propiedad recibira el valor undefined.

Metodos getters & setters

Los metodos getters son aquellos que sirven para obtener un valor, mientras que los setters son para modificarlos o definirlos.

Ej:
class perro extends animal {
    constructor(especie,edad,color,raza) {
        super(especie,edad,color);
        this.raza = null;
    }
    set setRaza(newName) {
        this.raza = newName;
}
    get getRaza() {
        return this.raza;
}
}

let perroA = new perro('perro', 5, negro, 'sharpey');
perroA.modificarRaza = 'Pastor Aleman';
console.log(perroA.getRaza);
OUTPUT:
Pastor aleman
 Nota: Los seteos no serán considerados como metodos así que deben ser empleados como propiedad.
